## How This File Is Used

### CEO reads at Phase 0:
- Check "Common Revision Requests" → anticipate what this project's user might want
- Check "Font Pairings — What Works/Failed" → pass to designer as "proven" vs "avoid"
- Check "Score Trends" → if a dimension is consistently weak, emphasize it to builder

### Designer reads at Phase 1:
- "Font Pairings" → prefer proven pairings for similar moods, avoid failed ones
- "Color Palettes" → avoid failed combinations, try variations of successful ones
- "Section Patterns" → borrow successful layout/motion combos for similar section types

### Builder reads at Phase 3:
- "Signature Elements — Approved/Rejected" → know what impresses vs what falls flat
- "Technique Effectiveness" → prefer high-scoring techniques
- "Score Trends — Weakest Dimension" → spend extra effort on historically weak areas

### CEO writes:
- After Phase 1 user review → font/color approval or rejection
- After Phase 3 batch review → section approvals, revision requests, signature reactions
- After Phase 5A final review → overall verdict
- Phase 6 retrospective → full project summary with scores, lessons, improvements
