The build pipeline is strict and non-negotiable:

1. Foundation docs BEFORE any code (content-brief -> design-brief -> page-plans -> motion-spec)
2. creative-design FIRST — without a design brief, no construction happens
3. 3D/WebGL ALWAYS — Tier 1 minimum (atmospheric shader or particle field)
4. Each section uses a DIFFERENT animation technique — never the same fade-up everywhere
5. Each project has UNIQUE identity — never reuse palette/typography/motion from another project
6. Quality chain runs post-build: validation-qa -> a11y -> seo -> responsive -> css -> perf
7. 0 BLOCKING + 0 CRITICAL = can deliver

**Why:** Projects without planning end up as templates. Design-first and content-first ensure every decision has a brief backing it.

**How to apply:** When starting any build task, check if docs/ exists and is complete. If not, create the docs first. Never improvise — if it's not in the brief, add it to the brief first.
