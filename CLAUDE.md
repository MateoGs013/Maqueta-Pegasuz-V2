## Design Philosophy — Anti-Slop

Every project must feel like it was designed by a senior creative director, not generated by AI.
Commit to a BOLD aesthetic direction and execute it with precision. The key is intentionality.

### NEVER produce (the "basic" test)

If you can describe a section as any of these, it's GENERIC and must be redone:

- "Centered title, paragraph, button" — the #1 sign of a basic page
- "Two equal columns with text left, image right" — a Bootstrap layout
- "Three cards in a row with identical styling" — template-tier
- "Fade-up on scroll" as the only motion — zero craft
- "Full-width image with text overlay, centered" — stock hero
- "Flat solid background with content floating in the middle" — no depth
- Uniform padding/gap on everything — no rhythm variation
- One hover state (opacity change) across the whole page — lazy
- Default easing (`ease`, `ease-in-out`) anywhere — unfinished
- Overused fonts: Inter, Roboto, Arial, system-ui — invisible choices
- Purple gradients, stock gradient backgrounds — AI-generated aesthetic
- Uniform border-radius on everything — no shape language
- Content that sits politely inside its container with no tension — boring

### ALWAYS produce (the "premium" test)

Every section must have at least 3 of these spatial qualities:

- **Overlap:** Elements crossing container boundaries, overlapping adjacent layers, negative margins, z-index choreography. Text over image. Decorative element bleeding across sections.
- **Bleed:** Content escaping its grid — images extending past the viewport edge, text columns that don't align to the global grid, asymmetric margins.
- **Tension:** Unequal proportions (70/30 not 50/50), vertical offsets between columns, elements positioned at unexpected coordinates, white space that feels intentional not leftover.
- **Depth:** Minimum 3 visual layers per section: (1) background atmosphere (gradient, noise, subtle color shift), (2) main content, (3) decorative/foreground elements (grain overlay, floating shapes, parallax-offset elements).
- **Scale contrast:** Dramatic size differences — 12px label next to 96px headline. Tiny detail next to massive hero image. Micro-element next to full-bleed visual.
- **Rhythm variation:** Sections alternate between generous whitespace (200px+ padding) and dense content. Never same padding on consecutive sections.
- **Compositional asymmetry:** Grid columns are 1.4fr/0.6fr not 1fr/1fr. Images offset by 40px from where you'd expect. Text aligned to unusual grid lines.

Also always:
- Distinctive font pairings with character — not "safe" choices
- Custom easing curves that feel physical: `cubic-bezier(0.16, 1, 0.3, 1)` not `ease`
- Rich near-blacks (`#0a0a0f`) or warm whites (`#fafaf7`) — never pure `#000` or `#fff`
- Micro-interactions: magnetic buttons, cursor reactions, hover reveals, focus states
- Scroll-driven narrative: pinned sequences, parallax layers, scrubbed timelines

## Quality Escalation Rules

### Composition (most important — this is what makes or breaks the design)
- If a section layout can be achieved with `text-align: center; max-width; margin: 0 auto` — it's too simple. Add spatial tension.
- No two sections should share the same grid structure. If S-Hero is 60/40 split, S-Features cannot also be 60/40.
- Every section needs a "spatial surprise" — one element that breaks expectation (bleeds, overlaps, offsets, oversized, or positioned unconventionally).
- Padding between sections should vary: hero might have 0 padding (full-bleed), next section 120px, next 200px. Never uniform.

### Typography
- Scale has 6+ distinct sizes with consistent ratio
- Display type in hero should be dramatically large — minimum `clamp(3rem, 8vw, 7rem)` for hero headlines
- Letter-spacing and line-height specified per type style, not globally
- Mix type weights dramatically: 300 body text next to 800 headlines

### Interaction density
- Minimum 3 distinct hover states per section (not just opacity changes)
- Every clickable element: hover + focus-visible + cursor change minimum
- At least 1 magnetic or physics-based interaction per page
- Links and CTAs must feel different from each other (not all the same button)

### Visual depth
- No section has fewer than 3 visual layers (background + content + decorative/foreground)
- Grain overlay alone does NOT count as a layer — it's a texture. A layer is a positioned element with its own spatial behavior.
- Background layers should respond to scroll (parallax, opacity shift, color transition)
- Decorative elements: subtle shapes, lines, dots, gradients that exist purely for composition

### Animation craft
- Every transition specifies its easing curve — never default `ease`
- Stagger values: 25-40ms for chars, 80-120ms for words, 150-200ms for elements
- Animations should have choreography — elements enter in a deliberate sequence, not all at once
- Never use `rgba(0,0,0,0.X)` for overlays — use the palette's darkest color with opacity

## Rules (apply to ALL teammates)
- Static first: hardcode all content. API wiring is a separate phase after visual approval.
- Only `transform` + `opacity` for animations — never width/height/top/left
- Parallax and scroll-linked animations: always `scrub: 0.5` — never `scrub: true`
- Spline scenes: dynamic import, `shallowRef`, `dispose()` on unmount, fallback image always present
- `prefers-reduced-motion` via `gsap.matchMedia()` — not manual `window.matchMedia` checks
- `autoAlpha` instead of `opacity` for all fade animations (also sets `visibility: hidden` at 0)
- `SplitText.create()` with `autoSplit: true`, `mask`, `aria: 'auto'` for text reveals
- `gsap.quickTo()` for mouse followers — never create new tweens on each mousemove
- `ScrollTrigger.batch()` for grid/list items — not individual triggers per element
- No consecutive sections with the same motion technique
- `var(--token)` for everything: colors, fonts, spacing, easing. No magic numbers.
- Register GSAP plugins once in `main.js` — never inside components
- Lazy route imports with `scrollBehavior` in router
- Images: `alt` + `width` + `height` + `loading="lazy"`
- No `axios` outside `src/config/api.js` · No HTTP outside `src/services/`
- No `will-change` preventive · No infinite decorative loops
- Semantic HTML with correct heading hierarchy
- `focus-visible` on all interactive elements

## GSAP Anti-Patterns (from greensock/gsap-skills official)
- Never put ScrollTrigger on child tweens inside a timeline — put it on the timeline itself
- Never use `scrub` and `toggleActions` together on the same ScrollTrigger
- Never nest `gsap.context()` inside `gsap.matchMedia()` — matchMedia creates its own context
- Never forget `immediateRender: false` when stacking multiple `from()`/`fromTo()` on the same property
- Never create ScrollTriggers in random order without `refreshPriority`
- Never leave `markers: true` in production
- Never skip `clearProps: 'all'` when CSS classes should take over after animation
- Call `ScrollTrigger.refresh()` after Vue `nextTick` when content changes dynamically

## Quality Checklist (per section)
- [ ] Semantic HTML, heading hierarchy, `aria-label`
- [ ] `var(--token)` everywhere, fluid type with `clamp()`
- [ ] Visual depth: minimum 2 layers (content + atmosphere). Overlap, shadow, gradient, blur, grain.
- [ ] Hover + `focus-visible` + magnetic effects on interactive elements
- [ ] Assigned motion technique with specific easing + timing + stagger values
- [ ] `gsap.matchMedia()` + `mm.revert()` cleanup (not manual `gsap.context`)
- [ ] `autoAlpha` for fades, `SplitText.create()` with `mask` + `aria` for text
- [ ] Responsive: 375px, 768px, 1280px, 1440px
- [ ] Grain/noise texture overlay on dark sections (2-4% opacity)
- [ ] No default easing anywhere — all cubic-bezier or GSAP named ease
- [ ] Asymmetric or intentional composition — not centered-everything

## Agent Teams Workflow

You are the LEAD. You orchestrate by spawning teammates with the prompts below.
Each teammate loads this CLAUDE.md automatically (including the Design Philosophy).
Teammates read files directly from disk — you do NOT extract/paste context inline.

**All teammates work inside `$PROJECT_DIR` (the new project directory, NOT maqueta).**
Tell each teammate the project path: `$PROJECT_DIR = C:\Users\mateo\Desktop\{slug}`.

### Phase 0: Discovery (Lead — do not delegate)

1. Parse the user's message. Extract: project type, name, mood, pages, references, constraints.
2. Ask ONLY what's missing (1 round, use options):
   - Pages: "Solo home" / "Home + About + Contact" / "Full site (5+)"
   - Brand: "Desde cero" / "Tengo colores/fonts"
   - Backend: "Estático" / "Conecta a API"
3. Compile identity card → show to user → confirm.
4. If reference URLs provided, capture with 4-pass sweep + auto-discovery.
   **IMPORTANT:** Capture script lives in maqueta, output goes to the PROJECT directory:
   ```bash
   # Single URL (auto-discovers internal pages, max 5):
   cd "$MAQUETA_DIR/scripts" && npm install --silent 2>/dev/null && node capture-refs.mjs "{url}" "$PROJECT_DIR/_ref-captures"
   # Batch (multiple URLs):
   cd "$MAQUETA_DIR/scripts" && npm install --silent 2>/dev/null && node capture-refs.mjs --batch "{url1}" "{url2}" --out "$PROJECT_DIR/_ref-captures"
   # Limit internal pages:
   cd "$MAQUETA_DIR/scripts" && npm install --silent 2>/dev/null && node capture-refs.mjs --max-pages 3 "{url}" "$PROJECT_DIR/_ref-captures"
   # Homepage only (no discovery):
   cd "$MAQUETA_DIR/scripts" && npm install --silent 2>/dev/null && node capture-refs.mjs --no-discover "{url}" "$PROJECT_DIR/_ref-captures"
   ```
   Where `$MAQUETA_DIR` = `C:\Users\mateo\Desktop\maqueta` and `$PROJECT_DIR` = the new project directory.
   Auto-discovers nav links and captures internal pages. Each page gets its own directory.
   Produces per page: `desktop/` + `mobile/` frames, `interactions/` screenshots, `manifest.json` v3.1.
   Site-level index at `_ref-captures/{domain}--index.json` maps all captured pages.
   Optionally spawn the `reference-analyst` subagent for structured analysis → `docs/reference-analysis.md`.

## Fallback: Subagents

If Agent Teams fails, the same workflow works with subagents via the Agent tool.
Agent definitions in `.claude/agents/` (designer.md, builder.md, polisher.md) have
frontmatter with scoped tools and model. Use them as regular subagents — same prompts,
same file references, same workflow. The only difference: subagents report back to you
instead of working as independent teammates.
