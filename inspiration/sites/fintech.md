## Que observar en este rubro

- **Trust signals:** seguridad, compliance badges, logos partners, numeros usuarios
- **Clarity:** productos complejos explicados con simplicidad visual
- **Dark mode:** prevalente en fintech tech-forward
- **Calculators/demos:** herramientas interactivas como engagement principal
- **Data visualization:** charts, graphs, dashboards como content visual

## Patrones comunes

| Pattern | Descripcion | Cuando usar |
|---------|------------|------------|
| Calculator hero | Herramienta interactiva como hero | Transparency como USP |
| Dashboard preview | Mockup del dashboard como feature showcase | Banking app SaaS |
| Gradient mesh bg | Background con gradientes animados | Hero, sections de impacto |
| Comparison table | Tabla interactiva de planes | Pricing, features |
| Card 3D mockup | Tarjeta bancaria con rotate 3D | Producto card/debit |
| Glassmorphism cards | Cards con backdrop-filter blur | Feature highlights |
| Animated numbers | Stats que countan on scroll | Social proof |

## Tecnicas de diferenciacion

1. **Transparency como UX** — mostrar exactamente cuanto cuesta, sin letra chica
2. **Product-as-hero** — la herramienta ES el hero, no una imagen
3. **Dark premium** — dark mode que transmite tech sophistication
4. **Data como storytelling** — numeros que countan, graphs que se dibujan
5. **Gradient como brand** — los colores en movimiento SON la identidad
